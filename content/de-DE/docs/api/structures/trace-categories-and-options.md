# TraceCategoriesAndOptions Objekt

* `categoryFilter` String - Ein Filter, um zu steuern, welche Kategoriegruppen nachverfolgt werden sollen. Ein Filter kann über ein optionales Präfix "-" verfügen, um Kategoriegruppen auszuschließen, die eine übereinstimmende Kategorie enthalten. Da sowohl als auch ausgeschlossene Kategoriemuster in derselben Liste enthalten sind, wird nicht unterstützt. Beispiel: `test_MyTest*`, `test_MyTest*,test_OtherStuff`, `-excluded_category1,-excluded_category2`.
* `traceOptions` String - Steuert welche Arten des tracing aktiviert werden sollen. Enthält eine Komma-Separierte Liste der folgenden Strings: `record-until-full`, `record-continuously`, `trace-to-console`, `enable-sampling`, `enable-systrace`, z.B. `'record-until-full,enable-sampling'`. Die ersten 3 Optionen sind Trace-Aufnahmemodi und schließen sich daher gegenseitig aus. Wenn in `traceOptions` mehr als ein trace recording Modus gesetzt wurde, dann hat der letzte Wert die höchste Priorität. Wen kein trace recording Modus gesetzt wurde, dann ist der recording Modus `record-until-full`. Die Ablaufverfolgungsoption wird zunächst auf die Standardoption zurückgesetzt (`record_mode` auf `record-until-full`, `enable_sampling` und `enable_systrace` auf `false`festgelegt), bevor Optionen angewendet werden, die von `traceOptions` analysiert wurden.
